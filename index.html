es6基础

let

  • 局部变量，会在在{}内有意义

  • 变量不可重复定义

  • 不会变量提升

  • 不影响作用域链

const

  • 和let类似
  • 声明的时候必须赋值
  • 赋值后就不可再进行赋值
  • 一般常亮进行大写，约定俗成
  • 定义的数组和对象，里面的值是可以进行修改的

解构赋值

ES6允许按照一定模式从数组和对象中提取值，对变量进行赋值

对象简化写法

箭头函数

  • this时静态的，始终指向函数声明时所在作用域下的下的this的值
  • 不能作为构造函数来实例化对象
  • 不能使用argment变量
  • 箭头函数不适合作为this有关的回调，事件回调，对象的方法
  • 简写
    • 当只有一个参数时，省略小括弧 b=>{}
    • 函数执行内容只有一行时，可以省略大括号，默认添加return , 比如： b=>b

函数默认值

  • 约定俗成，一般具有默认值的参数放后面
  • 与解构相结合

rest参数

  • 函数中，rest参数必须指定为…args，必须放所有参数的后面
  • args对象返回的是一个数组

扩展运算法

  • 语法：…，三个点号的形式
  • 将数组转换为以逗号分隔的参数序列

应用

  • 数组合并
  • 数组克隆
  • 将伪数组转换为真正的数组

Symbol

  • JavaScript第七种数据类型，表示独一无二的值，是一种类似字符串的数据类型
  • Symbol的值是唯一的，用来解决命名冲突的问题
  • Symbol值不能与其他数据进行运算
  • Symbol定义的对象属性不能使用for..in循环遍历，但是可以使用Reflect.ownKeys来获取对象的所有健名

迭代器

  • Iterator是一种接口，为各种不同的数据结构提供统一的访问机制。任何数据结构只有部署了Iterator接口，就可以完成遍历操作
  • ES6创造了一种新的遍历命令for…of循环
  • 原生具备Iterator接口的数据结构有
    • Array
    • Argument
    • Set
    • Map
    • String
    • TypedArray
    • NodeList
  • 原理：通过指针对象不断next遍历即可

JS-Promise分析

Promises/A+

Promise是基于Promises/A+标准实现的一套API

  • promise是一个有符合此标准的 then 方法的 object 或 function
  • promise有pending，fulfilled，rejected三种状态，开始时为pending，可转fulfilled和rejected，但是fulfilled和rejected状态无法再转变
  • then方法传入两个回调函数，监听promise的fulfilled和rejected两种状态，返回一个新的promise
  • catch方法捕获异常或者回调rejected状态
