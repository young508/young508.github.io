Promises/A+

Promise是基于Promises/A+标准实现的一套API

  • promise是一个有符合此标准的 then 方法的 object 或 function
  • promise有pending，fulfilled，rejected三种状态，开始时为pending，可转fulfilled和rejected，但是fulfilled和rejected状态无法再转变
  • then方法传入两个回调函数，监听promise的fulfilled和rejected两种状态，返回一个新的promise
  • catch方法捕获异常或者回调rejected状态

构造函数

  • 构造函数：constructor

then

  • 构造函数中入的resolver中两个回调方法我们可以理解为触发执行then中回调函数
  • 因为then的实现直接关系到resolver中的代码逻辑，并且由于是异步的，then也会先执行，所以我们先看then如何执行的
  • then的特点是传入两个回调函数，返回一个新的promise对象
  • 注意：这里有两条主线，一条是先执行构造函数的回调，再指定then回调
  • 二条是指定了then回调再指定了构造函数中的回调
  • then
  • 订阅者：subscribe
  • resolve方法是非常关键的方法
  • handleOwnThenable，处理自身返回值为promise类型的情况
  • fuifill和reject，无非是状态的改变和回调的监听
  • 发布者：publish

流程总结

  • 我们可以看到Promise通过订阅观察者模式，实现了回调监听
  • 分别对构造函数中的回调和then的回调的先后顺序做了不同的判断处理
  • 对then中的回调函数的返回着是否是Promise对象也做了不同的判断处理，如果返回值也是一个promise，那么实际上，该promis的作用就then返回的promise的作用
